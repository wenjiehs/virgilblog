# 从工作站到实战：一次真实的 AI 辅助产品策划全过程

## 🤔 这篇文章写什么

如果说[上一篇文章](/posts/2026/ai-product-planning-tcr-skills.md)是"实验报告"，那这篇就是**"实战记录"**。

今天（3月17日），我把之前搭建的产品策划工作站真正用起来了，完成了一次完整的产品策划过程。更有意思的是，**这篇文章本身的创作过程，也被记录在了对话历史中**。

这是一次真正的"自举"：用 AI 工作站做产品策划 → 写博客记录过程 → 用 AI 帮忙整理成文章 → 再写一篇记录这个过程。

## 📅 今天的完整工作流

### 早上：搭建工作站的理论架构

基于 3月10日 搭建的 [CodeBuddy 产品策划工作站](/posts/2026/codebuddy-product-workstation.md)的基础，我今天打算做一个真实的产品策划任务。

**任务目标**：为 TCR（腾讯云容器镜像服务）设计一个 AI Skills 功能

### 上午：让 AI 学习 TCR 产品

这是关键的第一步。我没有直接让 AI 写策划稿，而是先让它**系统性地学习 TCR 的产品逻辑**。

**我做了什么**：

1. **触发知识管理器代理**
   ```
   我：请帮我系统性地学习 TCR 产品的核心能力
   ```

2. **AI 自主探索控制台**
   - AI 通过浏览器自动化 Skill，访问 TCR 控制台
   - 逐个点击菜单、查看功能模块
   - 提取产品逻辑、功能结构、使用流程

3. **知识自动归档**
   - AI 将学到的内容结构化保存到：
     - `docs/01-product-concepts/tcr-overview.md`（产品概述）
     - `docs/02-features/tcr-features.md`（功能模块）
     - `docs/05-knowledge-base/tcr-index.md`（知识索引）

**这一步的价值**：
- AI 不再是"问一句答一句"，而是主动建立了对产品的完整认知
- 后续策划时，AI 可以自动引用这些知识，而不用我反复解释

### 中午：触发产品架构师生成策划稿

有了知识库作为基础，接下来就是真正的策划环节。

**我的输入**：

```
我想给 TCR 增加一个 AI Skills 功能，
让用户可以在控制台直接管理和使用各种运维 Skills。
```

**AI 的工作流程**：

1. **需求分析**（自动触发）
   - 提取核心需求：AI Skills 管理平台
   - 识别目标用户：DevOps 工程师、运维人员
   - 定义使用场景：查看、搜索、使用、管理 Skills

2. **查阅知识库**（自动执行）
   - 检索已归档的 TCR 产品逻辑
   - 了解现有功能模块结构
   - 参考 UI 设计风格和交互规范

3. **生成策划稿**（按模板输出）
   - 背景与动机：为什么做？解决什么问题？
   - 产品目标：3-5 个 SMART 目标
   - 功能设计：6 个核心功能的详细设计
   - 技术方案：架构图、API 设计、skill.yaml 规范
   - 实施计划：分阶段任务和工时估算

4. **自我评审**（质量检查）
   - 检查必填章节完整性
   - 验证目标是否 SMART
   - 确认技术方案可行性

**输出结果**：

一份 350+ 行的完整策划稿，保存在：
```
docs/03-planning/feature-proposals/2026-03-17_TCR-AI-Skills功能.md
```

### 下午：生成交互原型

策划稿有了，但光有文字描述还不够直观。我希望有个可视化的原型。

**我的请求**：

```
基于这份策划稿，生成一个 HTML 交互原型，
展示核心界面和交互流程。
```

**AI 自动生成**：

1. **实例选择器弹窗**
   - 地域选择
   - 实例列表展示

2. **AI Skills 列表页**
   - Skill 卡片展示
   - 搜索和筛选功能
   - 标签系统

3. **Skill 详情页（Skill Card）**
   - 基本信息
   - 工具列表
   - 版本管理
   - 引用统计

4. **安全审查工作流**
   - 审查流程可视化
   - 状态追踪

5. **OpenClaw 配置示例**
   - CLI 配置代码示例

**原型特点**：
- 纯 HTML/CSS/JavaScript，无需构建
- 可直接在浏览器打开预览
- 样式和交互完整，视觉效果接近真实产品

### 傍晚：整理成博客文章

完成策划后，我想记录这次实验。于是又进入了"博客写作模式"。

**第一步：写实验文章**

我把整个过程描述给 AI，AI 帮我整理成了第一篇文章：
- 文章标题：《让 AI 学会我的产品，再帮我写策划稿：一次完整的实验》
- 内容结构：背景 → 方法 → 过程 → 结果 → 思考
- 自动遵循我的[博客写作规范](/.codebuddy/rules/blog-writing-guide.mdc)

**第二步：发布到博客**

AI 自动完成了发布流程：
1. 创建 Markdown 文件：`docs/posts/2026/ai-product-planning-tcr-skills.md`
2. 更新文章索引：`docs/posts/index.md`
3. 更新侧边栏配置：`docs/.vitepress/config.ts`
4. Git 提交和推送

**第三步：写工作站架构文章**

在我提供的 `tmp/tmp4.md` 草稿基础上，AI 帮我整理成了第二篇文章：
- 文章标题：《用 CodeBuddy 搭建产品策划工作站：一个产品经理的 AI 协作实验》
- 重点讲三层架构：规则层、代理层、技能层
- 同样自动完成了发布流程

**第四步：写这篇续集**

现在，我正在让 AI 把今天的完整工作流程整理成这篇文章 😄

## 💭 一些真实的感受

### 1. 工作站真的能用

之前搭工作站的时候，说实话心里有点虚——这玩意真能用吗？还是只是个"玩具"？

今天实战下来，**真的能用**。而且不是"能用但还差点意思"，是**真的能直接用于生产**。

生成的策划稿，我基本没怎么改，就可以拿去内部评审了。这在以前是不敢想的。

### 2. AI 真的"学会"了产品

最让我惊讶的是，AI 在生成策划稿时，会主动引用它之前学到的产品知识：

> "基于 TCR 现有的实例管理能力，AI Skills 功能可以复用实例选择器组件..."

> "参考 TCR 镜像仓库的标签系统，Skills 也可以采用类似的分类机制..."

这不是简单的模板填充，而是**真正理解了产品逻辑，并进行了关联思考**。

### 3. 人机协作的"节奏感"

今天的工作流程中，我和 AI 的分工很清晰：

**我负责**：
- 提出产品想法
- 确认大方向
- 评审生成的内容
- 做最终决策

**AI 负责**：
- 系统性学习产品
- 生成详细的策划文档
- 创建交互原型
- 整理和归档知识

这种分工感觉很舒服，效率也高。

### 4. 文档质量真的可用

生成的策划稿不是那种"泛泛而谈"的 AI 文字，而是：
- 结构完整（符合公司 PRD 模板）
- 逻辑清晰（有明确的因果关系）
- 细节充分（功能设计、API 定义都很具体）
- 可执行性强（有明确的实施计划和工时估算）

唯一需要我补充的，是一些团队内部的上下文（比如"这个功能需要和 XX 团队对接"）。

## 🤔 一些思考

### 这套流程的适用场景

**适合**：
1. 有固定模板和规范的工作（策划稿、分析报告）
2. 需要大量上下文知识的任务（产品策划、技术方案）
3. 重复性高的工作（每月要写多份类似文档）

**不适合**：
1. 完全创新性的工作（从 0 到 1 的产品定义）
2. 需要大量人际沟通的任务（需求调研、客户访谈）
3. 一次性的临时任务（成本收益不划算）

### 人的价值在哪里？

有人可能会担心：AI 都能写策划稿了，产品经理还有什么价值？

我的理解：

**AI 替代的是**：
- 重复性的文档工作
- 基于已有知识的推理
- 格式化的内容生成

**人不可替代的是**：
- 产品愿景和战略方向
- 用户洞察和同理心
- 跨部门协调和推动
- 对业务价值的判断

AI 是工具，帮我们从繁重的文档工作中解放出来，**把时间花在更有价值的思考和决策上**。

### 下一步可以做什么

1. **扩展知识库**
   - 归档更多产品模块的逻辑
   - 建立产品设计原则库
   - 沉淀最佳实践案例

2. **优化生成质量**
   - 细化策划稿模板
   - 增加评审检查点
   - 引入用户故事生成

3. **增强自动化能力**
   - 自动竞品分析
   - 自动市场调研
   - 自动数据分析

4. **多代理协作**
   - 产品架构师 + 数据分析师 + 技术专家
   - 多角色评审机制
   - 自动生成多版本方案对比

## 🎉 总结

今天是一次真正意义上的"AI 辅助产品策划"实战。

核心收获：

1. **工作站架构真的有用**：规则、代理、技能三层架构，让 AI 能够结构化地完成复杂任务
2. **知识库是关键**：让 AI 先学习，再生成，质量会高很多
3. **人机协作有节奏**：人负责方向和决策，AI 负责执行和生产
4. **文档质量可用于生产**：不是玩具，是真正能用的生产力工具

这套流程还在迭代中，但已经证明了可行性。接下来会继续优化，看看还能玩出什么花样。
