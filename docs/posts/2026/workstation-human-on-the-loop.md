# 工作站的下一步：从 Human-in-the-Loop 到 Human-on-the-Loop

## 发现了一个瓶颈

上周我突然意识到一件事：我的 AI 产品策划工作站虽然功能越来越多，但每次要用它，都需要我主动说一句话。

- 想了解竞品动态？要我说"帮我查一下 GKE 最新的发布"
- 想沉淀月刊知识？要我把月刊文字粘贴进去说"帮我归档"
- 想看看知识库还缺什么？要我问"还有哪些待归档的内容"

本质上，**我是整个系统的瓶颈**。工作站的所有工作流，都需要经过我这个入口才能启动。

这就是 Human-in-the-Loop 的局限——人既是触发器，又是瓶颈。

---

## Human-in-the-Loop vs Human-on-the-Loop

这两个概念听起来像绕口令，但区别其实很直观：

**Human-in-the-Loop（人在循环中）**：
```
你发起 → AI 执行 → 你看结果 → 你再发起 → AI 再执行 ...
```

**Human-on-the-Loop（人在循环上）**：
```
AI 自主运转 → AI 遇到需要决策的节点 → 推送给你确认 → 继续运转
```

前者你是发动机，后者你是方向盘。

前者的问题是显而易见的：你不说话，什么都不发生。周末不开电脑，竞品出了大新闻，工作站毫不知情。你忙着开会，上周积累的想法没有人整理，就散落在各种对话记录里消失了。

后者的目标是：工作站持续运转，你只在关键节点介入。

---

## 我规划的三层演进

想清楚问题之后，我把演进路径拆成了三层。

### 第一层：让 Agent 定期主动工作

最简单的起点是定时任务。不需要复杂的 Agent 协作，只需要让现有的 Agent 按时间表自动触发。

我规划了三个任务：

**竞品动态周报**（每周一 09:00）

Agent 自己去扫各家竞品的官方博客和 What's New 页面，把新发布整理归档，顺手在 CHANGELOG 里记一条本周情报摘要。

周一上班，打开 CHANGELOG，5 分钟看完上周竞品有什么动静。从"我需要主动去查"，变成"它主动推给我"。

**知识库健康巡检**（每月月初）

自动扫描哪些内容标记了"待归档"但一直没动，哪些文档超过 60 天没更新，生成一份待办清单。

每月初花 10 分钟看一眼，决定优先补哪块内容。

**工作站周报**（每周五 17:00）

读 git log，整理本周新增了哪些知识、完成了哪些策划、有哪些新需求。生成一份简短的周报。

周五下班前瞄一眼，形成一周的工作记忆。

这三个任务的设计原则是：**Agent 产出，人来决策**。Agent 不决定要不要做，只是把信息整理好呈现出来，决策还是在我这边。

### 第二层：Agent 之间传递任务

第一层是"定时触发"，第二层是"事件触发"——一个 Agent 的输出触发另一个 Agent 的工作。

举几个场景：

**需求 → 策划自动流转**：`requirements-manager` 收到一个 P0 级客户需求后，自动建议"这个需求优先级很高，要不要生成策划稿？"。我确认，`product-architect` 开始工作，完成后需求卡片状态自动更新。

**竞品差距 → 想法记录**：竞品巡检发现某个 TKE 没有的能力，自动在 `06-ideas/` 生成一条想法记录，下次做规划时自然会看到。

**月刊导入 → 多文档联动**：导入产品月刊文字，`knowledge-manager` 不只更新一个功能文档，而是自动检查是否影响竞品差距分析、是否需要更新 Gartner 映射。

这一层的本质是：**减少我在 Agent 之间"传话"的工作**。

### 第三层：接入外部信息源

前两层解决的是"触发"和"联动"，第三层解决的是"感知"——工作站自己知道外面发生了什么。

设想的做法是接入竞品官方博客、K8s 版本发布等信息源，定时抓取，让 `competitive-analyst` 判断相关性，把值得关注的内容自动归档。

信息从"需要我导入"，变成"自动流入"。

## 目前的进展

第一层任务已经设计好了，三个 Automation 的 Prompt 都写完了，在工作站的 `workstation-ops/EVOLUTION.md` 里。

今天在另一个 IDE（WorkBuddy）上把 Automation 跑起来了。下周一就会自动触发第一次竞品动态巡检，看看效果。

还顺手做了几件事：

**重新设计了 CHANGELOG**，加了"思考与收获"这一节，每个版本记录一段阶段性认知。以前 CHANGELOG 只是文件列表，现在它开始有"人味"了——半年后回看，能看到认知在哪里发生了变化。

**整理了工作站目录**，把运营文件（TIPS、EVOLUTION、周报、健康报告）统一放进 `workstation-ops/` 文件夹，和 TKE 业务文档分开。这件事做完感觉清爽了很多。

**加了文档位置 Rule**（`alwaysApply: true`），避免 Agent 把工作站运营文件放到业务知识库里——这个问题今天就踩到了好几次。

## 一点感受

工作站用到现在，最深的感受是：**AI 工具的价值，很大程度上取决于你给它设计了什么样的结构**。

同样是"让 AI 帮你整理竞品"，一次性对话和有结构化知识库的工作站，效果天差地别。

但结构本身也会成为新的瓶颈。当工作站有了 6 个 Agent、10 多个知识目录、几十个文档之后，光维护它就需要不少精力——这和"用 AI 节省精力"的初衷有点背道而驰。

所以 Human-on-the-Loop 对我来说不只是个技术方向，更是一种设计哲学：工具应该适应人的工作节奏，而不是反过来要求人去伺候工具。

第一层任务跑起来之后，看看实际效果。
