# 零游戏开发经验，一个下午用 AI 复刻了魔兽经典塔防"绿色循环圈"

## 🤔 起因：一个周末下午的念头

今天下午闲着没事，突然想到一个问题：**AI 现在到底能帮我做到什么程度的项目？**

之前我用 AI 做过博客、做过工作流工具、做过产品策划——但这些多少都在我的舒适圈里。我想试试一个完全不熟的领域。

然后我想到了"绿色循环圈"。

如果你玩过魔兽争霸3的 RPG 地图，你一定知道这个名字。这是一个经典的塔防地图——怪物在一个闭合跑道上无限循环奔跑，玩家在跑道两侧建造防御塔。不同于普通塔防"怪物走直线"的模式，绿色循环圈的怪物是**永远不会走出地图**的，它们就在圈里跑，直到被打死或者数量爆炸游戏结束。

我的想法很简单：**能不能用 Web 技术把它复刻出来？**

说"复刻"可能有点夸张。更准确地说是：**在一个下午的时间内，看看 AI 能帮我把这个游戏做到什么程度。**

## 🎯 我的起点

先说说我的"游戏开发经验"：**零。**

- ❌ 没写过任何游戏
- ❌ 不懂游戏引擎
- ❌ 不知道什么是游戏循环、帧率、碰撞检测
- ❌ 对 Canvas/WebGL 只停留在"听说过"的程度

我有的只是：
- ✅ 能写点 TypeScript
- ✅ 会用 Vite 搭项目
- ✅ 玩过绿色循环圈（虽然是十几年前的事了）
- ✅ 一个 AI 编程助手

核心问题：**这些够吗？**

## 🛠️ 实际过程：一个下午的时间线

### 13:00 - 需求调研（30分钟）

第一步不是写代码，而是让 AI 帮我做"需求调研"。

我对 AI 说了一句：*"了解下绿色循环圈这个魔兽 RPG 塔防地图，我想要复刻一个 1:1 还原的 Web 游戏。"*

AI 做了一件让我觉得很靠谱的事——它没有直接开始写代码，而是先去搜索了大量资料，包括百度百科、贴吧攻略帖、游戏评测等，然后给我整理了一份完整的游戏机制总结：

| 维度 | 说明 |
|---|---|
| 地图结构 | 方形闭合跑道，怪物无限循环 |
| 失败条件 | 场上怪物>100 或 BOSS超时 |
| 经济系统 | 金钱+木材，可互换 |
| 塔防类型 | 16种塔，含光环/英雄成长/秒杀 |
| 克制矩阵 | 7种攻击×8种护甲 |
| 波次 | 50波+BOSS+隐藏关+无尽 |

然后它问了我三个关键决策问题：复刻哪个版本？技术栈选什么？美术怎么处理？

这让我意识到一个有意思的事：**AI 在做项目规划时，思路其实挺清晰的。**

### 13:30 - 技术选型和设计（20分钟）

经过讨论，我们确定了方案：

- **引擎**：Phaser 3 + TypeScript（AI 推荐的，说是成熟的 2D 游戏框架）
- **构建**：Vite
- **美术**：先用几何图形占位，后期替换
- **策略**：单人模式先行，核心玩法优先

AI 给出了一份详细的设计文档，包括文件结构、系统划分、分阶段交付计划。说实话，这份设计文档的质量让我有点意外——它不像是 AI 临时编的，更像是一个有经验的游戏开发者的方案。

当然，后来实际开发中发现了不少问题（后面会说），但作为起点已经很不错了。

### 13:50 - Phase 1：核心骨架（40分钟）

AI 一口气创建了大约 15 个文件，搭建了完整的项目骨架：

```
src/
├── config/          # 塔/怪物/波次数据
├── systems/         # 路径/经济/波次/网格管理
├── entities/        # 塔/怪物/弹道实体
├── scenes/          # 菜单/游戏/结算场景
└── main.ts          # 入口
```

第一次 `npm run dev` 就跑起来了，TypeScript 零编译错误。

屏幕上出现了一个绿色的方形地图，中间有一条灰色的闭合跑道。点击底部的商店按钮选一个塔，放到地图上，按 N 键——一群彩色小圆点开始在跑道上跑。

**虽然看起来很简陋，但……它确实在动了。** 怪物在跑，塔在射击，弹道在飞，怪物死了会给钱。

坦白说，这个时刻我有点激动。十几年前玩的那个游戏，现在在浏览器里跑起来了（虽然丑了十倍）。

### 14:30 - Phase 2：修 Bug + 完善交互（50分钟）

跑起来之后，问题就来了。这也是这次实验中我觉得最有价值的一个观察——**AI 写的代码第一版一定会有 Bug，但它修 Bug 也很快。**

主要问题：
1. **弹道不动** — AI 创建了弹道对象但忘了在主循环中调用 `update()`。这是个很典型的"知道概念但忽略细节"的问题
2. **点击塔没反应** — 事件处理的优先级搞混了
3. **商店按钮超出屏幕** — 10个塔的按钮太多，布局没算好
4. **英雄塔成长公式数组越界** — `level=0` 时访问了 `upgrades[-1]`

每个问题我只需要描述现象，AI 就能定位到代码并修复。大部分是一次修好，个别的需要两三轮。

这个阶段还加了不少东西：暂停功能、数字快捷键选塔、Shift 连续建造、下一波倒计时、建造粒子特效、BOSS 震屏等。

### 15:20 - Phase 3：完整塔防内容（60分钟）

这是工作量最大的阶段。AI 再次搜索了大量原版攻略数据，然后重写了三个核心配置文件：

- **40种怪物**（原版50波全还原），含飞行/隐形/魔免/毒免特性
- **16种塔**（多重/弹射/重击/毒/减速/枪兵/腐蚀/光环/防空/混乱/破坏/火海/魔幻/秒杀/侦查）
- **完整50波配置**，每波对应真实的怪物组合

同时把护甲系统从 6 种扩展到 8 种（加了加强甲和普通甲），完善了魔兽争霸3的原版克制矩阵。

这个阶段的体验是：**数据配置是游戏开发中最枯燥但最重要的部分。** AI 在这方面帮了大忙——手动配置40种怪物的属性、50波的组合，如果让我自己来，光查资料就得一天。

### 16:00 - 英雄塔系统（40分钟）

原版绿色循环圈有一个很有特色的机制：开局随机获得一个英雄成长塔，不同英雄有不同的技能和属性体系。

这部分是整个项目中最复杂的系统：

- 12种英雄塔（火魂/小黑/剑神/电法/水魔/风神/战神/兽王/恶魔之子/寒冰/幻影/神牛）
- 每个英雄4个技能（区分主动/被动）
- 三维属性系统（力量/敏捷/智力）+ 成长曲线
- 经验/升级/技能点分配
- 主动技能 CD + 自动施放
- 暴击/多重箭/链式闪电等战斗效果

开局时会弹出卡牌选择界面，从3个随机英雄中选1个。选完后英雄塔自动放置，通过击杀怪物获得经验升级。

这个系统让我见识到了 AI 处理复杂业务逻辑的能力——12个英雄的配置数据+技能效果+战斗计算+UI面板，它基本上一次性就写出了能跑的版本。当然，后面还是需要好几轮迭代来完善细节（比如属性加点窗口会被误关闭的 Bug）。

### 16:40 - P0 核心机制补全（30分钟）

做了个整体 Review 之后发现，虽然塔和怪物都有了，但几个核心机制还没实现：

1. **飞行单位** → 所有塔都能打 → 防空塔没意义
2. **隐形单位** → 直接可见 → 侦查塔没意义
3. **魔免/毒免** → 没实际效果
4. **光环塔** → 加攻/加速光环不生效
5. **腐蚀降甲** → 只做了额外伤害，没有真正降低护甲

这5个一起修，涉及 Enemy、Tower、GameScene 三个核心文件。AI 同时处理了所有修改，编译一次通过。

### 17:15 - 塔效果 + 资源UI + 隐藏关/无尽（30分钟）

最后一波更新：
- 重击塔暴击、弹射塔弹射、秒杀塔秒杀
- 木材显示 + 金→木/木→人口 交换按钮
- 50波通关后自动进入隐藏关（10波强敌）
- 隐藏关 + 40PF 后进入无尽模式（怪物无限增强）

## 🤔 几个真实的反思

### AI 做对了什么

**1. 项目规划能力出乎意料**

从需求调研→技术选型→文件结构→分阶段计划，AI 给出的方案是有体系的。它选择 Phaser 3 作为游戏引擎是个正确的决定——如果让我自己选，我可能会从纯 Canvas 开始，然后在物理碰撞那里卡住半天。

**2. 数据配置是它的强项**

40种怪物、16种塔、50波配置、12种英雄的技能数据——这些如果手动写，至少要大半天。AI 能快速查阅资料并生成结构化的配置数据，这个效率提升是数量级的。

**3. 修Bug能力很强**

描述现象 → 定位问题 → 修复代码，通常一次搞定。AI 比人类更擅长在大量代码中快速定位问题，因为它能同时"看到"所有文件。

### AI 做错了什么

**1. 第一版代码总有遗漏**

最典型的是弹道没有在游戏主循环中被 update——AI 写了 `Projectile.update()` 方法，也写了 `Tower` 里创建弹道的代码，但忘了在 GameScene 的 update 循环中调用它。这说明 AI 在"全局一致性"上还是有盲点的。

**2. 事件系统的理解不够深**

Phaser 的事件冒泡机制和 DOM 不同，AI 第一次处理"点击信息面板按钮不应该穿透到地图"这个问题时，用了 `stopPropagation`——但在 Phaser 中 scene 级别的 input 监听是无法被子对象 stop 的。最后是用坐标范围检测解决的。

**3. 需要人来把控"好不好玩"**

AI 能实现功能，但不太能判断"这个数值好不好玩"。比如重击塔的暴击率设多少、无尽模式每波增强多少比例，这些都需要人去感受和调整。

### 对我个人的意义

这是我第一次做游戏。说实话，整个过程中最让我兴奋的不是最终成果，而是**那种"我居然能做游戏"的感觉**。

作为一个产品经理，我一直在写需求文档、画流程图、和开发沟通。AI 正在模糊"想"和"做"之间的界限——你不需要精通游戏引擎，不需要理解渲染管线，只要你清楚"我想做什么"，AI 就能帮你搭出来。

当然，这不意味着开发者不重要了。恰恰相反，这次实验让我更深刻地理解了**好的架构设计、细节把控和性能优化有多重要**——这些是 AI 还不太擅长的，也是专业开发者的核心价值。

## 🎮 想试玩？

项目地址在 GitHub（项目名 WebGreenLoop），用 `npm install && npm run dev` 就能跑起来。

操作提示：
- 底部商店选塔 → 点地图放置
- `1-0` 快捷选塔 | `U` 升级 | `S` 出售
- `N` 下一波 | `P` 暂停 | `Space` 加速
- `H` 查看攻击/护甲克制表 | `M` 移动英雄

如果你也想试试用 AI 做个游戏，我的建议是：
1. **选一个你熟悉的游戏**——你比 AI 更懂"好玩"是什么意思
2. **先跑通核心循环**——一个能射击的塔+一个能走的怪就够了
3. **数据配置让 AI 来**——这是它效率最高的部分
4. **不要追求完美**——先丑着跑起来，再慢慢打磨
